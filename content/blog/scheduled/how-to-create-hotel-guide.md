「館内案内をちゃんと作りたいけど、何から始めればいいか分からない」。
この状態は、ホテル・民泊の現場でよくあります。

実際には次のような“その場しのぎ”で回っているケースも少なくありません。

- とりあえず紙で対応
- PDFをなんとなく配布
- スタッフが口頭で補足

この運用を続けると、情報が分散し、現場負担が増えやすくなります。

## ① 共感

館内案内が機能しない理由は、ツール以前に「設計」が曖昧なことにあります。

- 載せる情報が担当者ごとに異なる
- 順番にルールがなく読み手が迷う
- 更新時に何を直すべきか判断しづらい

まず必要なのは、作成前に全体構成を決めることです。

## ② 問題提起

よくある失敗は、構成を決めずに作り始めることです。

- 思いついた情報をそのまま載せる
- 情報の順番がバラバラ
- 必要な情報が抜ける
- ゲスト目線になっていない

つまり問題は、作り方そのものより「設計不足」です。  
構成がないまま作ると、どの手段でも使われにくい案内になってしまいます。

## ③ 解決策比較（比較表）

館内案内の作成方法を、設計しやすさと運用面で比較すると次の通りです。


| 作成方法    | 特徴      | メリット          | デメリット      |
| ------- | ------- | ------------- | ---------- |
| 紙で作成    | 従来型     | 手軽            | 更新不可・情報不足  |
| PDF作成   | デジタル化初期 | 配布しやすい        | 見づらい・修正が面倒 |
| Webページ  | 自由度高い   | 表現が豊富         | 作成難易度が高い   |
| Infomii | 構造型     | 誰でも作れる・更新しやすい | 初期設計が必要    |


ポイントは、「何で作るか」より「どう設計するか」です。

## ④ ベストな方法

館内案内は、次の順番で作ると失敗しにくくなります。

### 1. 全体構成を先に決める

最初に骨組みだけ決めます。

基本構成の例:

- 施設基本情報
- Wi-Fi情報
- 朝食・営業時間
- 館内設備
- チェックアウト案内
- よくある質問

まずは項目を確定し、詳細は後から埋めるのが効率的です。

### 2. 情報をブロック単位で分ける

1画面に詰め込みすぎず、情報を分解して配置します。

例:

- 見出し
- テキスト
- 画像
- ボタン

情報のまとまりを作るだけで、視認性が大きく改善します。

### 3. ゲストの行動順で並べる

運営者目線ではなく、ゲストが使う順番で並べるのが基本です。

例:

- チェックイン直後 -> Wi-Fi
- 滞在中 -> 館内設備
- 翌朝 -> 朝食
- 出発前 -> チェックアウト

行動順に沿うと、必要情報への到達が速くなります。

### 4. 導線を設計する

作るだけでは使われません。見る場所まで設計します。

例:

- 客室QR -> 館内案内
- フロント -> よくある質問
- エレベーター -> 施設案内

「どこで見せるか」を決めることで利用率が上がります。

### 5. 小さく作って改善する

最初から完璧を目指さず、段階的に拡張します。

例:

- まずWi-Fiだけ
- 次に朝食案内
- 問い合わせが多い項目を追加

運用しながら改善するほうが、現場に定着しやすくなります。

## ⑤ CTA

館内案内は「作ること」より、「使われる設計」にすることが重要です。

もし今、

- なんとなく作っている
- 情報がバラバラ
- 更新が止まっている

この状態なら、一度「構成」から見直すだけでも改善が進みます。
