# Week11 Execution Tickets (Monetization Tightening)

## Goal
ホテル特化SaaSとして、導入初週の成果を維持しながらPro課金と継続利用を安定化させる。

## Day1
- [x] 施設規模別オンボーディング完了率を自動比較
- [x] 初回公開から2本目公開までの時間を短縮する導線を追加
- [x] テンプレ複製から公開までのショートカット導線を追加

## Day2
- [x] LPヒーロー訴求を業態別に最終固定しA/Bを停止
- [x] LPのCTA遷移率をデバイス別（SP/PC）で可視化
- [x] 事例セクションの閲覧率をスクロール深度で比較

## Day3
- [x] 編集画面に「公開後チェック」パネルを追加
- [x] 問い合わせ先未入力時の修正導線を1クリック化
- [x] 画像最適化未実施ブロックを優先表示

## Day4
- [x] 休眠通知の送信タイミングを施設稼働時間で最適化
- [x] 通知文面の勝ちパターンを自動選択
- [x] 再開後7日継続率を通知チャネル別に比較

## Day5
- [x] Pro転換阻害要因の改善タスクを自動提示
- [x] 請求・カード更新完了後の再開導線を強化
- [x] 課金イベント失敗時の運用手順をUIに固定表示

## Day6
- [x] 週次レポートに「実行済み改善数」を追加
- [x] 重大アラートのみ即時通知するしきい値を調整
- [x] 運用センターの表示優先順位をKPI基準で再配置
