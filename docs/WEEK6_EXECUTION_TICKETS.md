# Week6 Execution Tickets (Hotel Conversion Stabilization)

## Goal
ホテル向けの商談化後オンボーディングを安定させ、公開継続率とPro継続率をさらに引き上げる。

## Day1
- [ ] 業種別LP（ビジネス/リゾート/温浴）をURL単位で分離
- [ ] LPごとのCVRを運用センターで比較表示
- [ ] 流入チャネル別に推奨CTAを自動表示

## Day2
- [ ] 登録直後に「初回公開ウィザード」を表示
- [ ] 公開前チェックのエラー説明に具体例を追加
- [ ] 公開後共有モーダルにLINE/メール直送ボタンを追加

## Day3
- [ ] テンプレ品質スコア（情報充足度）を可視化
- [ ] 低スコアテンプレに不足項目補完ガイドを表示
- [ ] テンプレごとの利用回数ランキングを表示

## Day4
- [ ] 施設規模（客室数）ごとの推奨構成テンプレを追加
- [ ] ノードマップに「導線切れ」検知を追加
- [ ] 子ページの戻る導線をカスタム文言対応

## Day5
- [ ] 公開ページのCLS/INP計測を追加
- [ ] 画像最適化未対応URLを一括検出
- [ ] 遅いページの改善優先度を自動スコア化

## Day6
- [ ] 休眠予兆を3日/7日/14日で段階通知
- [ ] 再開施策別に7日/14日継続率を比較
- [ ] 週次レポートをSlack/メール送信可能にする

## Day7
- [ ] Week6 KPIレビュー（CVR/公開率/Pro転換/継続）
- [ ] 標準化施策をテンプレ初期値へ反映
- [ ] Week7実装チケットを作成
