# Week9 Execution Tickets (Activation Automation)

## Goal
ホテル向け導入後の初回成果をさらに短縮し、運用開始1週間以内の公開継続率を引き上げる。

## Day1
- [ ] LPの業態別勝ち訴求を固定配信し、勝ち以外の文言を非表示化
- [ ] LP CTAクリック率をセクション別（hero/sticky/bottom）で比較
- [ ] LP導線のSNS流入別に最適CTAを自動提案

## Day2
- [ ] 初回公開ウィザードにQR配布証跡（日時）を追加
- [ ] ウィザード再開リンクをメールテンプレへ連携
- [ ] Stepごとの離脱理由入力を追加

## Day3
- [ ] 施設タイプ別テンプレTop3の再評価ジョブ（手動実行）を追加
- [ ] 低利用テンプレ改善案の自動提案文を業態別に出し分け
- [ ] テンプレ選択→公開中央値を週次推移で表示

## Day4
- [ ] 公開前チェックの修正候補を優先度順で並び替え
- [ ] ワンクリック修正を複数項目まとめて適用
- [ ] 公開前スコア低下時に理由を自動表示

## Day5
- [ ] 画像最適化の実行前後サイズ差を可視化
- [ ] 速度改善施策の履歴比較をページ単位で表示
- [ ] 施策別工数見積もりと期待改善値を併記

## Day6
- [ ] 休眠通知の反応（既読/未反応）をチャネル別で集計
- [ ] 再開勝ち導線を業態別に初期選択
- [ ] 週次レポートの自動出力フォーマットを固定化

## Day7
- [ ] Week9 KPIレビュー（LP CVR / 公開率 / Pro転換 / 継続）
- [ ] Week10チケットを作成
- [ ] Week9リリースノートを作成
