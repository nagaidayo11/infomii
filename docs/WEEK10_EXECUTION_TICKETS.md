# Week10 Execution Tickets (Scale & Retention)

## Goal
ホテル導入の初回成果を維持しつつ、継続率と紹介流入を同時に伸ばす。

## Day1
- [ ] 施設規模別の初回公開ガイドを分岐表示
- [ ] 公開後48時間の定着アクションを自動提案
- [ ] テンプレ採用率が高い業態の導線を優先表示

## Day2
- [ ] LPの導入事例セクションに業態タグ別フィルタを追加
- [ ] CTA文言を流入チャネル別に自動最適化
- [ ] LPスクロール離脱地点ヒートマップを運用センターに表示

## Day3
- [ ] 編集画面のブロック追加パネルに「おすすめ構成」を追加
- [ ] 未入力項目の一括補完テンプレを提供
- [ ] 公開直前チェックに「問い合わせ導線」必須判定を追加

## Day4
- [ ] 休眠兆候ユーザーの再訪予測スコアを追加
- [ ] 通知チャネルの勝ちパターンを施設タイプ別に自動選択
- [ ] 通知送信後24時間の反応率を週次比較で可視化

## Day5
- [ ] Pro転換時の阻害要因アンケートを追加
- [ ] 料金ページのFAQを離脱理由上位順で再構成
- [ ] 請求・カード管理導線の完了率を運用センターに追加

## Day6
- [ ] 週次レポートに「改善アクション実行率」を追加
- [ ] 失敗イベントの再発防止チェックリストを固定表示
- [ ] 運用センターのアラート優先度表示を調整

## Day7
