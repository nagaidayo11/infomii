# Template Block Order Guide (Category Best Order)

This guide defines the recommended card order for each marketplace category.
The order is designed to reduce guest confusion and shorten time to QR publish.

## Common principles

- Put arrival/decision-critical information first.
- Keep each block focused on one action.
- Place reassurance/rules near the end (FAQ, notice, checklist).
- Keep core flow visible in first 4-6 blocks.

## business

Recommended order:

1. `hero`
2. `kpi` (quick facts)
3. `wifi`
4. `schedule` / `restaurant`
5. `laundry` / facilities
6. `checkout`
7. `faq` / `taxi` / `nearby`
8. `emergency` (if needed)

Why: speed and clarity for short-stay business guests.

## resort

Recommended order:

1. `hero`
2. `gallery`
3. `spa` / key experience
4. `menu` / dining
5. `schedule`
6. `nearby` / `map`
7. `pageLinks`
8. `quote` / `faq`
9. `checkout` / `emergency`

Why: lead with value/experience, then operational details.

## ryokan

Recommended order:

1. `hero` or `welcome`
2. `notice` (custom/rules)
3. `spa` / bath guidance
4. `restaurant` / `menu`
5. `steps` / flow
6. `schedule`
7. `nearby` / `map`
8. `checkout`
9. `faq` / `emergency`

Why: communicate hospitality and etiquette before logistics.

## airbnb

Recommended order:

1. `hero`
2. `steps` (self check-in)
3. `checklist` (house rules)
4. `wifi`
5. `nearby` / living info
6. `map`
7. `checkout`
8. `emergency`
9. `faq` / `notice`

Why: self-service flow and risk prevention are most important.

## guide

Recommended order:

1. `hero`
2. `nearby`
3. `map`
4. `pageLinks`
5. `taxi` / transport
6. `schedule`
7. `notice`
8. `faq` / `quote`
9. `emergency`

Why: action-oriented city movement with destination confidence.

## inbound

Recommended order:

1. `hero` or `welcome`
2. `notice` (language/reassurance)
3. `pageLinks`
4. `map`
5. `wifi`
6. `steps`
7. `checkout`
8. `emergency`
9. `faq` / `taxi`

Why: reduce anxiety and increase wayfinding clarity for first-time guests.
