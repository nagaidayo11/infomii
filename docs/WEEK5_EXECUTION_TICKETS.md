# Week5 Execution Tickets (Hotel Monetization Sprint)

## Goal
ホテル向けの商談化率とPro継続率をさらに改善し、導入後14日で「更新が回る状態」を標準化する。

## Day1
- [x] LPヒーローの訴求をホテル業務別（チェックイン/温浴/朝食）に出し分け
- [x] CTA文言を3パターンでAB比較できるようにする
- [x] LPの実例セクションに「導入施設コメント」を追加

## Day2
- [x] 新規登録後の初回導線を「テンプレ選択」固定に最適化
- [x] 初回公開までのチェックリストを編集画面右カラムにも表示
- [x] 公開時にスタッフ共有を自動提案するモーダルを追加

## Day3
- [x] テンプレを業務シーン別に再編（フロント/客室/温浴/レストラン）
- [x] テンプレごとに推奨公開タイミング（朝/昼/夜）を表示
- [x] テンプレ保存（お気に入り）をチーム共有可能にする

## Day4
- [x] 公開ページのFAQブロックに「折りたたみ表示」を追加
- [x] 長文セクション向けに目次ジャンプブロックを追加
- [x] 子ページの戻る導線をテンプレ化（ホテル向け既定デザイン）

## Day5
- [x] 速度低下アラート時に「修正候補ブロック」を自動提示
- [x] 画像URL入力時にサイズ/形式の事前チェックを追加
- [x] SLA未達ページの優先修正リストを運用センターに追加

## Day6
- [x] 休眠検知を7日/14日で二段階化し通知内容を分岐
- [x] 再開施策別の7日継続率を比較（テンプレ再開/下書き再開）
- [x] 管理者向け週次レポート文面を自動生成

## Day7
- [x] Week5 KPIレビュー（登録率/公開率/Pro転換率/継続率）
- [x] 勝ち施策をデフォルト導線に昇格、弱い施策を停止
- [x] Week6実装チケットを作成
