# App Shell UI — Expo Go 検証チェックリスト

`docs/APP_SHELL_UI_MOTION_PROMPTS.md` Phase A–F 実装後の手動確認用です。

**前提:** `?client=app` または Expo WebView、`npm run dev:lan` + 実機

---

## タブ・遷移（Phase C）

- [ ] ホーム → 作品 → テンプレートでコンテンツが横にスライドする（または reduced motion でフェードのみ）
- [ ] タブバー上部のアクセント線が選択タブに追従する
- [ ] 同一タブ内の遷移（例: 設定内）ではタブスライドが走らない

## 作品（Phase B + F）

- [ ] カード上部タップでエディタ、公開スイッチのみ切替
- [ ] 公開 ON/OFF 成功時に画面上部トースト（緑系）
- [ ] 失敗時に赤系トースト（`alert` ではない）
- [ ] ヘッダー「更新」で一覧再読み込み
- [ ] FAB「+」に押下フィードバック

## エディタヘッダ（Phase D）

- [ ] 公開スイッチが作品一覧と同型
- [ ] 保存状態文言が切り替わるときフェード
- [ ] ⋯ メニューが下からボトムシートで開く

## ホーム（Phase E）

- [ ] AI ブロックがヒーローカード内に収まっている
- [ ] 統計のラベルが薄いメタ文字、分析行がタップ可能
- [ ] 最近の作品がコンパクトカード

## テンプレート（Phase E）

- [ ] カテゴリがセグメント風（選択＝アクセントソフト）
- [ ] カード押下で軽いスケール、プレビュー画像表示

## 設定（Phase E + F）

- [ ] チェックボックス・ラジオ行が 44px 前後
- [ ] 最下部に赤い「ログアウト」ブロック

## 共通（Phase F）

- [ ] 主要リンク・ボタンに押下感（`ui-pop-tap` / `app-pressable`）
- [ ] iOS 設定「アニメーションを減らす」ON で激しいスライドが抑えられる
- [ ] ルート遷移中にトップ progress bar が見える

---

## 既知のスコープ外

- Pull-to-refresh（Expo WebView では未実装 → 「更新」ボタンで代替）
- ネイティブ haptic（`AppSwitch` に TODO コメントのみ）
