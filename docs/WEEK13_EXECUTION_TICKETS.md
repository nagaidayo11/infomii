# Week13 Execution Tickets (Monetization Scale)

## Goal
ホテル特化の導線を維持しながら、紹介経由の新規獲得と有料継続を同時に伸ばす。

## Day1
- [ ] LPの業態別LPを3パターンに分離し、検索流入向けFAQを最適化
- [ ] CTA文言を「30秒登録」軸で統一し、チャネル別補助文を最終調整
- [ ] LP読み込み速度のボトルネック画像を上位5件改善

## Day2
- [ ] 事例セクションを「課題→解決」構成へ再編集
- [ ] 成果指標（時間削減/問い合わせ削減）を数値カードで固定表示
- [ ] ログイン遷移前に業態選択を入れてテンプレ推薦精度を上げる

## Day3
- [ ] 編集画面に「ホテル向け完成テンプレパック」クイック適用を追加
- [ ] 公開前チェックで危険項目（リンク切れ/未公開遷移）を強制停止
- [ ] 画像圧縮ガイドを実行チェック付きに更新

## Day4
- [ ] Pro転換導線の比較表を「損失回避」訴求へ変更
- [ ] past_due復旧の自動リマインド間隔を最適化
- [ ] 請求導線完了率を日次で可視化

## Day5
- [ ] 休眠通知を施設タイプ別シナリオに分岐
- [ ] 通知文面のAB候補を管理者のみで再テスト可能にする
- [ ] 通知後の再公開率を14日で追跡

## Day6
- [ ] 運用センターに「今週の最優先3施策」自動生成を追加
- [ ] 重大アラートにWebhook再送チェックを追加
- [ ] 週次レポートの自動送信テンプレを整備

## Day7
- [ ] Week13 KPIレビュー（LP→登録 / 公開完了 / Pro転換 / 14日継続 / 紹介）
- [ ] Week14チケットを作成
- [ ] Week13リリースノートを作成
