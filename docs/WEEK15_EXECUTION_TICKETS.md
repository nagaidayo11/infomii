# Week15 Execution Tickets (Expansion Readiness)

## Goal
ホテル導線の勝ち筋を維持しつつ、案件増加時の公開品質と継続率を同時に守る。

## Day1
- [ ] LPの「導入までの不安」解消ブロックを業態別に最適化
- [ ] CTAのクリック後ファネルをref/kw単位で分解可視化
- [ ] LP主要セクションのLCP悪化要因を自動検出

## Day2
- [ ] テンプレ作成後の初回公開率をテンプレ単位で比較表示
- [ ] テンプレカードに「想定運用人数」メタを追加
- [ ] 失敗しやすいテンプレ項目を初期値として補完

## Day3
- [ ] 編集画面に「公開直前チェックリスト（必須5項目）」を固定表示
- [ ] 画像/リンク/法務の警告を優先度順に並べ替え
- [ ] 子ページ導線不足を自動提案

## Day4
- [ ] Pro訴求を利用状況別（件数/公開頻度）に出し分け
- [ ] checkout離脱ユーザーへの再開導線を1クリック化
- [ ] 請求・カード管理画面への遷移率を計測

## Day5
- [ ] 休眠予兆（更新停止・公開減）ユーザーへ再開テンプレ提示
- [ ] 施設タイプ別の復帰メッセージを自動選択
- [ ] 14日継続率を「新規/既存」で分割表示

## Day6
- [ ] 運用センターに「今週のボトルネック」自動抽出を追加
- [ ] 重大アラートに復旧完了チェックを追加
- [ ] 週次レポートに「改善実行率」を追加

- [ ] Week16チケットを作成
