## 2. Notion×Canva 級 UI/UX 設計

### 2.1 両者の強みの統合

| 要素 | Notion | Canva | Infomii 統合方針 |
|------|--------|-------|------------------|
| **編集体験** | スラッシュコマンド、インライン編集 | ドラッグ＆ドロップ、キャンバス | 両方対応：スラッシュでブロック追加 + キャンバス配置 |
| **学習コスト** | 低い（テキスト中心） | 直感的（ビジュアル） | テンプレートから開始でゼロから作らせない |
| **レスポンス** | 即時反映 | リアルタイムプレビュー | 編集即プレビュー、オートセーブ |
| **モバイル** | アプリ対応 | モバイル編集 | ゲスト向けは375pxプレビュー必須 |
| **コラボ** | リアルタイム | チーム | Phase 2 で検討 |

### 2.2 エディタ統合アーキテクチャ（現状整理）

**現状**: 3系統のエディタが存在
1. **Editor2** (`/editor/[id]`) — カード型、Notion風、メイン
2. **page-editor** — ブロック型、ドラッグソート
3. **SaasEditor** (`/editor/saas/[id]`) — Canva風、Rnd で自由配置

**推奨**: **Unified Editor** に統合
- 左: ブロックライブラリ（Notion風カテゴリ）
- 中央: **ハイブリッドキャンバス**
  - デフォルト: 縦スクロールのブロックリスト（Notion風）
  - オプション: 自由配置モード（Canva風）— 設定で切り替え
- 右: プロパティパネル（選択ブロックの設定）
- 上部: スラッシュコマンド、検索、プレビュー切替

### 2.3 具体的なUX改善項目

#### A. スラッシュコマンド強化（Notion風）
- `/` でブロック追加メニュー（既存 SlashCommandMenu を拡張）
- 検索可能: `/wifi` → WiFiブロック、`/朝食` → 朝食ブロック
- 最近使ったブロックを上部表示

#### B. インライン編集
- テキストブロック: クリックで即編集、フォーカス時にツールバー表示
- 画像: クリックでアップロード/URL、ドラッグでリサイズ

#### C. ドラッグ＆ドロップ
- ブロックの並べ替え: `@dnd-kit` でスムーズに
- 左パネルからキャンバスへドラッグで追加
- ブロックハンドル（⋮⋮）でドラッグ

#### D. キーボードショートカット
- `Cmd+S`: 保存（既存オートセーブと併用）
- `Cmd+Z` / `Cmd+Shift+Z`: Undo/Redo
- `Cmd+D`: ブロック複製
- `Backspace`（空ブロック）: 削除
- `Enter`: 下に新規テキストブロック

#### E. スマホプレビュー
- 375px 幅のリアルタイムプレビュー（既存 MobilePreview を強化）
- デバイスフレーム、縦横切替
- 編集とプレビューの同期

#### F. テンプレート体験
- 初回: テンプレート選択 or 白紙
- テンプレート適用後、ブロックをそのまま編集可能
- テンプレートギャラリーの充実（業種別）

## 3. 技術アーキテクチャ

### 3.1 スタック（現状維持）

- **Frontend**: Next.js 16, React 19, Tailwind 4, Zustand, Framer Motion
- **Backend**: Vercel, Supabase (Auth, DB, RLS)
- **決済**: Stripe
- **AI**: OpenAI API（既存: generate-cards-from-url, translate, info-chat）

### 3.2 データモデル（schema.sql ベース）

```
hotels → hotel_memberships (多対多)
       → pages (Editor 2.0)
       → subscriptions (Stripe連携)

pages → cards (Editor2 のカード型)
```

**注意**: `informations` と `pages` が併存。段階的に `pages` に統一する方針を推奨。

### 3.3 エディタ状態管理

- **Zustand** で一元管理（既存 useEditor2Store, usePageEditorStore, useSaasEditorStore）
- 統合後: `useUnifiedEditorStore` に集約
- Undo/Redo: `zustand/middleware` の `subscribeWithSelector` + 履歴スタック
