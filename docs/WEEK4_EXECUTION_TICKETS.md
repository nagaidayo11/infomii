# Week4 Execution Tickets (Hotel Scale Sprint)

## Goal
ホテル導入の再現性を高め、運用の属人化をなくして継続課金率を安定化する。

## Day1
- [ ] 導入初日の運用手順を1画面に固定（初期設定チェックリスト）
- [ ] 公開ページの多言語テンプレ（JP/EN）を追加
- [ ] QR掲示用の印刷テンプレ（A4）を追加

## Day2
- [ ] 施設タイプ別（ビジネス/リゾート/温浴）に推奨テンプレを自動提示
- [ ] テンプレ選択後の不足項目を「必須/推奨」で表示
- [ ] 初回公開までの離脱箇所を可視化

## Day3
- [ ] スタッフ招待の承認待ちリマインドを追加
- [ ] 共有導線のクリック率（LINE/メール）を計測
- [ ] 共有文面テンプレを業務シーン別に拡張

## Day4
- [ ] Pro導線の表示ロジックを季節要因（繁忙期）対応へ拡張
- [ ] 課金FAQに「繁忙期のみ利用」導線を追加
- [ ] 決済失敗時の再試行案内を強化

## Day5
- [ ] LCP低下の原因ブロック（画像/ギャラリー）を自動診断
- [ ] 速度低下ページの改善提案を自動表示
- [ ] テンプレごとの速度目標（SLA）を定義

## Day6
- [ ] 休眠再開導線の完了率を施設タイプ別に比較
- [ ] 休眠通知テンプレのAB（短文/詳細）を計測
- [ ] 再開後7日継続率を運用センターに追加

## Day7
- [ ] Week4 KPIレビュー（LP→登録率 / 公開完了率 / Pro転換率 / 継続率）
- [ ] 弱い施策を停止し、勝ちパターンを標準化
- [ ] Week5実装チケットを作成
