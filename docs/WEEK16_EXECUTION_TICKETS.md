# Week16 Execution Tickets (Revenue Discipline)

## Goal
ホテル向け収益導線を崩さず、登録後7日以内の公開完了率とPro転換率を同時に引き上げる。

## Day1
- [ ] LPファーストビューの業態別勝ちコピーを再検証
- [ ] CTAクリック後の離脱理由をref別に分類
- [ ] 主要LPのLCP悪化ページを自動通知

## Day2
- [ ] テンプレ上位3件を施設規模別に固定表示
- [ ] 初回公開率が低いテンプレに改善ヒントを表示
- [ ] テンプレプレビューに「公開後の運用タスク」要約を追加

## Day3
- [ ] 編集画面の公開前チェックで重大項目を先頭固定
- [ ] 子ページ未接続時の修正アクションをワンクリック化
- [ ] 画像最適化候補の一括提案を業態別に最適化

## Day4
- [ ] Pro訴求を閲覧数/更新頻度/公開枠でスコア化
- [ ] checkout再開導線の完了率をref別で可視化
- [ ] 請求・カード管理導線の離脱ステップを表示

## Day5
- [ ] 休眠復帰テンプレを施設タイプ別で自動提案
- [ ] 通知文面の勝ちパターンを曜日別に最適化
- [ ] 新規/既存ユーザーの継続率差分をアラート化

## Day6
- [ ] 運用センターに「今週の収益リスク」セクションを追加
- [ ] 重大アラートの復旧完了ログを必須化
- [ ] 週次レポートに「改善未実行項目」を追加

## Day7
- [ ] Week16 KPIレビュー（LP→登録 / 初回公開 / Pro転換 / 14日継続 / 請求完了）
- [ ] Week17チケットを作成
- [ ] Week16リリースノートを作成
