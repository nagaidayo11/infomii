# Week3 Execution Tickets (Hotel Revenue Focus)

## Goal
ホテル向けの導入決定率とPro継続率を改善し、受注後7日以内の成果体験を最大化する。

## Day1
- [ ] ホテル専用LPを1本化（業種分岐を削減）
- [ ] ヒーローCTAを「無料登録」と「ログイン」の2導線で固定
- [ ] 実際の公開ページ事例を3件掲載（チェックイン / 温浴 / 館内設備）

## Day2
- [ ] テンプレ選択画面に「おすすめ順（用途別）」を追加
- [ ] テンプレカードに想定運用シーン（フロント/客室/館内）を明示
- [ ] 初回公開までの手順をテンプレ作成直後に固定表示

## Day3
- [ ] 公開直後のQR配布導線を強化（コピー文面3種）
- [ ] スタッフ向け共有導線（LINE/メール）をワンクリック化
- [ ] 公開ページに「最終更新時刻」の視認性強化

## Day4
- [ ] Proアップグレードの表示条件を閲覧規模連動で最適化
- [ ] 課金FAQをホテル運用質問ベースに差し替え
- [ ] checkout再開導線のクリック率を運用センターに表示

## Day5
- [ ] 公開ページのLCPをテンプレ別に比較表示
- [ ] 画像サイズが重いブロックに警告表示を追加
- [ ] 速度低下ページの自動抽出（運用センター）

## Day6
- [ ] 休眠検知（7日停止）から再開までの完了率を計測
- [ ] 再開導線のAB（テンプレ再開 vs 下書き再開）を比較
- [ ] 休眠施設向けの管理者通知テンプレを追加

## Day7
- [ ] Week3 KPIレビュー（LP→登録率 / 公開完了率 / Pro転換率）
- [ ] 効果が低い導線を停止し、上位2施策に開発集中
- [ ] Week4実装チケットを作成
