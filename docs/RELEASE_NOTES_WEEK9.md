# Release Notes - Week9

## 概要
Week9では、初回公開の失敗要因を減らしつつ、運用センターで改善判断を回せるように自動化と可視化を強化しました。

## 主な変更
- 初回公開ウィザードにQR配布証跡（日時）と離脱理由入力を追加
- ウィザード再開導線をメールテンプレから呼び出せるよう改善
- 施設タイプ別テンプレTop3の手動再評価導線を追加
- 低利用テンプレの改善提案文を業態別に出し分け
- テンプレ選択→公開中央値の週次推移を表示
- 公開前チェックを優先度順に並び替え、一括修正を追加
- 公開前スコア低下時の理由表示を追加
- 画像最適化の実行前後サイズ差・速度施策のページ別比較を追加
- 工数見積もり（S/M/L）と期待改善値を併記
- 休眠通知の反応（既読/未反応）をチャネル別集計
- 再開導線を業態別に初期選択
- 週次レポート出力フォーマットを固定化
- Week9 KPIレビューカード（LP CVR / 公開率 / Pro転換 / 継続）を追加

## 技術ノート
- `ops/health` APIの `week9Preview` に以下を追加
- `templatePublishTrend4w`
- `restartDefaultPathByFacility`
- `dormancyReactionByChannel`
- `trackDormancyNoticeReaction` を追加し、通知反応イベントを記録
- 編集画面の公開前チェックで優先度ソート・一括修正・スコア低下理由表示を実装
- `docs/WEEK10_EXECUTION_TICKETS.md` を新規作成
