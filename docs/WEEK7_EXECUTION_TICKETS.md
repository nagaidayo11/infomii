# Week7 Execution Tickets (Hotel Monetization Optimization)

## Goal
商談化後の運用継続を維持しながら、LP→登録→初回公開→Pro転換の全体CVRを改善する。

## Day1
- [ ] LPヒーロー文言を業態別にABテスト（ビジネス/リゾート/温浴）
- [ ] LP比較で勝ちパターンを自動表示
- [ ] LP下部CTAを業態別に差し替え

## Day2
- [ ] 新規登録後の初回公開ウィザードを3画面に分割
- [ ] ウィザード離脱ログを計測
- [ ] 1画面目完了率をダッシュボードに追加

## Day3
- [ ] テンプレ上位3件を施設タイプ別に固定表示
- [ ] 低利用テンプレに「改善候補」タグを追加
- [ ] テンプレ選択から公開までの中央値を表示

## Day4
- [ ] 公開前チェックのエラー種別ごとに発生率を可視化
- [ ] エラー修正導線をワンクリック化
- [ ] 公開直前の推奨修正を自動提案

## Day5
- [ ] 公開ページのLCP/INP/CLS改善施策を自動提案
- [ ] 画像最適化の一括変換オペレーションを追加
- [ ] 優先度スコアに改善工数（S/M/L）を追加

## Day6
- [ ] 休眠3/7/14日通知の配信ログを実装
- [ ] 再開施策別（テンプレ/下書き/公開確認）の勝ちパターンを固定化
- [ ] 週次レポートに「今週の最優先3施策」を追加

## Day7
- [ ] Week7 KPIレビュー（LP CVR / 初回公開率 / Pro転換 / 14日継続）
- [ ] Week8チケットを作成
- [ ] リリースノートを作成
